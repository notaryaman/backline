GR6 README

How to open and evaluate the medium-fi prototype.

What Backline is

Just-in-time coaching for retail fulfillment associates when the floor gets hectic.

Device to use

Phone-sized viewport. The prototype is laid out as a handheld support tool for active pick runs.

How to open it

Open the GR6 prototype and use the launch buttons for the simple, moderate, or complex tasks.

Hard-coded content

Inventory, supervisor responses, timer states, and location hints are illustrative sample data.

How to run the three tasks

Simple: Dashboard to item detail, missing item support, choose a recovery action, then confirm.
Moderate: Dashboard to interruption mode, help customer, review resume card, then continue the run.
Complex: Dashboard to repeated issue prompt, escalation card, waiting state, supervisor reply, then apply recommendation.

Known limitations

Inventory data is hard-coded to a limited set of sample orders.
Supervisor responses are pre-authored rather than live or generated.
Timer changes are simulated and do not update in real time.
Store map and location hints are illustrative only.
